ZONE 1 — EXTERNAL SYSTEM (Command Sender)
🖥 External SCADA System
Command / update originator
Sends commands or data updates
destined for the EBI Screen / TMS.

Examples:
• Schedule override / timetable update
• Train hold / door command
• Emergency broadcast trigger
• Configuration push
→
📤 Message Publication
External system publishes an AES-256-GCM encrypted JSON message
RabbitMQ / MQTT target:
Exchange: amq.topic

Routing key: scada.tms.commands
OR direct to Kafka:
Topic: scada.tms.inbound
Payload format (encrypted):
// Wire format (same as outbound)
Byte 0–11 : IV (12 bytes, random)
Byte 12–end: AES-256-GCM ciphertext
+ 16-byte GCM auth tag

// After decryption — JSON command:
{
 "messageType": "SCADA_TMS_CMD",
 "timestamp": "2026-04-13T09:00Z",
 "command": "HOLD_TRAIN",
 "trainId": "guid-6250",
 "platformId": "PL2201"
}
🔒 Security: Same AES-256-GCM key shared via SCADA_AES_KEY env var. External system must encrypt using the same key. GCM tag provides tamper detection — any modified message is automatically rejected by the Bridge.
Encrypted message delivered via RabbitMQ/MQTT or directly to Kafka
ZONE 2 — KAFKA (Inbound Staging & Replay Buffer)
⚡ Kafka Topic: scada.tms.inbound
Inbound command buffer — mirrors the outbound pattern
Why Kafka for inbound?
✔ Replay if TMS is temporarily offline
✔ Audit trail — all inbound commands retained
✔ Multiple consumers (Bridge + logger + monitor)
✔ Decouples external system from Artemis uptime
Route option A — via RabbitMQ first:
RabbitMQ (scada.tms.commands)
→ Bridge Route 3
→ Kafka (scada.tms.inbound)
→ Bridge Route 4
→ Artemis
Route option B — direct to Kafka:
Kafka (scada.tms.inbound)
→ Bridge Route 3
→ Artemis
Bridge consumes from Kafka inbound topic

Complete Bidirectional Flow Summary
⬇ OUTBOUND (existing) — EBI → External
1. TMS publishes XML → Artemis topics
2. Bridge Route 1: Artemis → XML→JSON + Encrypt → Kafka tms.scada.encrypted
3. Bridge Route 2: Kafka → RabbitMQ tms.scada.pas
4. External SCADA consumes from RabbitMQ/MQTT
⬆ INBOUND (new) — External → EBI
1. External SCADA publishes encrypted JSON → RabbitMQ scada.tms.commands
2. Bridge Route 3 NEW: RabbitMQ → Kafka scada.tms.inbound
3. Bridge Route 4 NEW: Kafka → Decrypt + JSON→XML → Artemis TMS.Commands
4. TMS receives XML → forwards to EBI Screen
