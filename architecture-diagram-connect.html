INTERNAL

Pink Line TMS

TMS VM · 10.4.0.23 · namespace: pinkline · own & operate
Everything Pink Line owns and runs. Source XML messages from TMS, transforms them (XML→JSON, AES encrypt), stores them in Kafka, and ships to the shared hub. Pink Line controls deployment, scaling, and restarts.

EXTERNAL

SCADA

SCADA VM · 10.4.0.25 · namespace: scada · 3rd-party
Owned by the SCADA team — black box from Pink Line's perspective. Pink Line drops messages into RabbitMQ; RabbitMQ's MQTT plugin forwards them to SCADA. Responses come back the same way.
A
🐰 RabbitMQ
SHARED HUB
Integration broker · AMQP ↔ MQTT plugin
  • AMQP 30672 NodePort (TMS-side ⇄)
  • MQTT 31883 NodePort (SCADA-side ⇄)
  • mgmt 31567 NodePort
  • exchange amq.topic
  • queue scada.tms.alarms.queue
  • routing keys → MQTT topics (auto-translation by plugin)
B
🎯 SCADA API
EXTERNAL
SCADA application · Python · Flask · Paho MQTT
  • HTTP 30891 NodePort (dashboard + /api/status)
  • subscribes MQTT tms/scada/pas
  • publishes MQTT scada/tms/alarms
  • AES-256-GCM decrypt with shared key
  • RSAE protocol: UpdateAlarm · KeepAlive · SendAllAlarms · GetAllAlarms

▶ ◀ DATA FLOWS (after toggle flip)

  • OUTBOUND: TMS → Artemis → Connect-source → Kafka tms.raw → Camel (XML→JSON+AES) → Kafka tms.scada.encrypted → Connect-sink → RabbitMQ AMQP → MQTT plugin → SCADA
  • INBOUND: SCADA → MQTT → RabbitMQ queue → Connect-source → Kafka scada.tms.raw → Camel (decrypt + JSON→XML opt) → Kafka scada.tms.processed → Connect-sink → Artemis

⚠ DEAD-LETTER QUEUES (5)

  • dlq.connect.tms-artemis-source Kafka · 14d
  • dlq.connect.tms-rabbitmq-sink Kafka · 14d
  • dlq.connect.scada-rabbitmq-source Kafka · 14d
  • dlq.connect.scada-artemis-sink Kafka · 14d
  • DLQ.kafka-bridge Artemis queue · existing

🛡 SECURITY · RELIABILITY

  • AES-256-GCM both directions (authenticated)
  • 12-byte IV + 16-byte GCM tag per message
  • Single AMQP wire — easy to firewall + monitor
  • Connect: state in _connect-* Kafka topics — pod restart safe
  • Camel: supervised routes, exp backoff 5s→60s
  • Default-off toggles ⇒ safe rollback at any time
