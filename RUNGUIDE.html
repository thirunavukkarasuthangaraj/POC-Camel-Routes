Phase 1 — Install Requirements (one time only)
1
Install Java 17
The app is built in Java. You need Java 17 or higher.
Download: https://adoptium.net  →  choose Java 17 LTS → Windows x64 .msi
After install, open CMD and verify:
java -version
Expected Output
openjdk version "17.0.x" ...
OpenJDK Runtime Environment Temurin-17...
❌ If you see "'java' is not recognized" → Java not installed. Install it first.
2
Install Maven 3.8+
Maven builds the Java code and downloads all dependencies.
Download: https://maven.apache.org/download.cgi → apache-maven-3.9.x-bin.zip
Extract to C:\maven
Add C:\maven\bin to Windows PATH environment variable
Verify:
mvn -version
Expected Output
Apache Maven 3.9.x ...
Java version: 17.x.x
3
Install Docker Desktop
Docker runs the local servers: Artemis, Kafka, Zookeeper, RabbitMQ.
Download: https://www.docker.com/products/docker-desktop → Windows installer
After install, start Docker Desktop and wait for it to say "Running" in the taskbar.
Verify:
docker info
Expected Output
Server Version: 24.x.x
Containers: 0
Running: 0
❌ If you see "Cannot connect to Docker daemon" → Docker Desktop not started. Open Docker Desktop app and wait.

Phase 10 — Verify Encryption (end-to-end decrypt test)
12
Understand what SCADA receives and how to decrypt
SCADA API connects via MQTT to tms/scada/pas topic and receives encrypted byte[]. It decrypts using the same AES-256-GCM key.
WIRE FORMAT (what is inside each Kafka/MQTT message):
Byte 0-11  (12 bytes) : IV (random, different every message)
Byte 12-end           : Ciphertext + 16-byte GCM authentication tag

Example (185 bytes total):
a3 f2 c8 4d 11 9e 6b 2a 55 f0 d1 7c  ← 12-byte IV
b7 4a 22 ff ... e5 6c 90 aa bb cc dd ee ff 00 11 22 33 44 55 66 77  ← ciphertext + GCM tag
SCADA DECRYPT CODE (Java — see DecryptExample.java):
// 1. Get IV from first 12 bytes
byte[] iv = Arrays.copyOfRange(payload, 0, 12);

// 2. Get ciphertext (rest of bytes)
byte[] ciphertext = Arrays.copyOfRange(payload, 12, payload.length);

// 3. Load same AES key from environment variable
byte[] keyBytes = Base64.getDecoder().decode(System.getenv("SCADA_AES_KEY"));
SecretKey key = new SecretKeySpec(keyBytes, "AES");

// 4. Decrypt — GCM tag verified automatically
Cipher cipher = Cipher.getInstance("AES/GCM/NoPadding");
cipher.init(Cipher.DECRYPT_MODE, key, new GCMParameterSpec(128, iv));
String json = new String(cipher.doFinal(ciphertext), StandardCharsets.UTF_8);

Troubleshooting — Common Problems
!
Problem Checklist
❌ "Docker not running" / "Cannot connect to Docker daemon"
→ Open Docker Desktop application
→ Wait until taskbar shows "Docker Desktop is running"
→ Try again
❌ "Port 61616 already in use"
→ Another Artemis/ActiveMQ is running
→ Run stop-local.bat first, then try again
→ Or: docker ps → find the container → docker stop <id>
❌ "Connection refused to localhost:9092"
→ Kafka not ready yet (takes ~20 seconds after start)
→ Wait 30 seconds and retry
→ Check: docker ps to confirm test-kafka shows "Up"
❌ Bridge app starts but no messages appear in logs
→ Did you run send-test-xml.bat? In LOCAL mode Artemis is empty at start.
→ Run send-test-xml.bat AFTER the app says "Started KafkaBridgeApplication"
→ In PRODUCTION: TMS server publishes automatically, no extra step needed
❌ Tests FAIL with "NoSuchMethodError"
→ Java version mismatch
→ Run: java -version — must show version 17 or higher
→ If lower, uninstall and reinstall Java 17
❌ RabbitMQ login fails (http://localhost:15672)
→ Use: tms_bridge / testpass123 (NOT guest/guest — local Docker uses tms_bridge)
→ If container not yet ready, wait 15 seconds and refresh
Summary — Full Message Journey
∑
Complete Flow in One View
1. TMS Server → Publishes XML to Artemis topics (4 topics)
2. Artemis :61616 → Stores XML until bridge reads it
3. Camel Route 1 → from("artemis:TMS.PISInfo") — polls every 500ms
4. XmlToJsonProcessor → Jackson XmlMapper → POJO → Jackson ObjectMapper → ICD JSON string
5. EncryptProcessor → Random 12B IV + AES-256-GCM encrypt JSON → byte[] [IV|cipher|GCM-tag]
6. Kafka :9092 → Stores encrypted byte[] in topic: tms.scada.encrypted
7. Camel Route 2 → from("kafka:tms.scada.encrypted") — consumer group: scada-bridge-consumer
8. RabbitMQ :5672 → AMQP publish → exchange: amq.topic, routing key: tms.scada.pas
9. MQTT :1883 → RabbitMQ MQTT plugin converts → MQTT topic: tms/scada/pas
10. SCADA API → Subscribes MQTT → receives byte[] → AES-256-GCM decrypt → ICD JSON → platform display
